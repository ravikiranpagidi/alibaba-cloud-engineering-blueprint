# Enterprise Landing Zone

## Purpose

A landing zone is the prepared cloud foundation where teams can deploy workloads safely and consistently.

## Building Blocks

- Identity and access baseline with RAM, MFA, roles, and access reviews.
- Network baseline with VPCs, vSwitches, route tables, NAT, private connectivity, and inspection.
- Security baseline with KMS, Cloud Firewall, Security Center, ActionTrail, Cloud Config, and SLS.
- Operations baseline with monitoring, incident routing, backup, and DR.
- Cost baseline with tags, budgets, allocation, and cleanup processes.
- Platform baseline with Terraform or ROS modules and approved deployment pipelines.

## Adoption Model

1. Establish central governance.
2. Define workload onboarding checklist.
3. Build shared network and logging.
4. Pilot a low-risk workload.
5. Expand to production with reviews.
6. Continuously improve controls based on incidents and audits.
