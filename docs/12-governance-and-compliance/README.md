# Governance And Compliance

    ## Purpose

    Create controls for resource organization, auditability, compliance, cost ownership, and enterprise adoption.

    ## What To Learn

    - Resource groups
- Tags
- Policies
- Cloud Config
- ActionTrail
- Audit logs
- Cost allocation
- Landing zone basics
- Multi-account controls

    ## Hands-On Practice

    - Create a tagging policy
- Write a compliance checklist
- Design a landing zone

    ## Interview Prompts

    - How do you prove who changed a resource?
- What controls should exist before teams deploy production workloads?

    ## Production Lens

    For every design, ask: What can fail? Who can access it? How is it monitored? How is it backed up? How much will it cost? How will it be cleaned up or decommissioned?
