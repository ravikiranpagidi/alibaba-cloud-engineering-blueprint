# Migration Patterns

    ## Purpose

    Plan cloud migrations using practical workload, data, network, security, and operations patterns.

    ## What To Learn

    - Rehost
- Replatform
- Refactor
- Retire
- Retain
- Database migration
- Hybrid connectivity
- Cutover planning

    ## Hands-On Practice

    - Build a migration wave plan
- Design rollback
- Plan Oracle to Alibaba Cloud migration

    ## Interview Prompts

    - How do you migrate with minimal downtime?
- How do you handle dependencies between workloads?

    ## Production Lens

    For every design, ask: What can fail? Who can access it? How is it monitored? How is it backed up? How much will it cost? How will it be cleaned up or decommissioned?
