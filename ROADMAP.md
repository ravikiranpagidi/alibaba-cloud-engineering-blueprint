# Roadmap

This roadmap keeps the repository practical, polished, and useful for learners and enterprise teams.

## Phase 1: Foundation

- Build complete repository structure.
- Add core README, contribution guide, security policy, and code of conduct.
- Add learning notes for cloud fundamentals, core Alibaba Cloud services, and service comparisons.
- Add beginner labs and project templates.

## Phase 2: Hands-On Depth

- Expand labs with screenshots, CLI commands, Terraform variants, and validation scripts.
- Add cost estimates with clear assumptions.
- Add cleanup automation for safe lab teardown.
- Add architecture diagrams as Mermaid and exported PNG/SVG assets.

## Phase 3: Enterprise Architecture

- Add landing zone designs for single-account, multi-account, and regulated environments.
- Add governance controls for tags, budgets, Cloud Config, ActionTrail, RAM, KMS, and SLS.
- Add migration playbooks for web apps, databases, analytics platforms, and hybrid networks.
- Add ADR examples for production decisions.

## Phase 4: Data And AI Platform

- Expand OSS data lake, MaxCompute, DataWorks, Realtime Compute for Apache Flink, Hologres, and AnalyticDB guides.
- Add medallion architecture examples.
- Add AI-ready platform patterns using PAI, feature pipelines, model serving, vector search, RAG, monitoring, and AI governance.

## Phase 5: Community And Quality

- Add GitHub issues for beginner-friendly contributions.
- Add diagram review checklist.
- Add automated Markdown link checks.
- Publish GitHub Wiki pages.
- Add release notes and changelog entries for major content milestones.

## Maintenance Principles

- Prefer official Alibaba Cloud documentation for service facts.
- Keep labs runnable by beginners with clear cleanup steps.
- Mark assumptions for cost, region, quota, and service availability.
- Avoid vendor marketing language. Explain tradeoffs.
- Keep diagrams simple enough to discuss in interviews.
